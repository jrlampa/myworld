# CONTRIBUIÇÃO

1. **Padronização de Código**
   - Use `npm run lint` e `npm run format` para JS/TS.
   - Use `black .` para Python.
2. **Commits**
   - Mensagens claras e descritivas.
3. **Testes**
   - Adicione ou atualize testes ao implementar novas funcionalidades.
4. **Documentação**
   - Atualize `README.md` e `ARQUITETURA.md` conforme necessário.
5. **Pull Requests**
   - Descreva claramente as mudanças e relacione issues relevantes.

---
> **Mantenha este guia atualizado conforme o fluxo de trabalho evoluir.**
